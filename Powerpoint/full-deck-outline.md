# Que 전체 기능 발표 — 목차·개요 설계 (v1, 2026-07-13)

> 용도: ①전체 기능판 PPT의 페이지 설계도 ②영상 제작 시 그대로 대본이 되는 구조.
> 각 툴 장은 **왜 써야 하는가 → 어떻게 쓰는가 → 잘 썼을 때 효과** 3단 고정 + 예시 영상 자리.
> ✅=영상 확보됨(Powerpoint/videos/), 🎬=추가 촬영 필요(콘티 포함).

## 전체 구조 (4부 · 21장)

| 부 | 장 | 내용 |
|---|---|---|
| 1부 도입 | 1~3 | 표지 · 왜 Que인가(Before/After) · 전체 지도 |
| 2부 리듬 | 4~6 | 하루 · 일주일 · 수시(변경/긴급) — 툴이 아니라 "생활" |
| 3부 툴별 | 7~18 | 12개 툴, 장마다 왜/어떻게/효과 + 영상 |
| 4부 마무리 | 19~21 | 숫자로 보는 효과 · 지켜지는 규칙(안심) · 시작하기 |

발표 시간 가늠: 장당 1~1.5분 → **25~30분** (영상 포함). 짧게 쳐낼 땐 3부에서 9~12장(성과·결제·관리자·MCP)을 부록으로 밀면 **18분**.

## 1부 — 도입

### 1. 표지
- Que — 감시하지 않고, 병목을 드러낸다. 8인 팀의 Company OS.
- que.griff.co.kr · 2026-07-20(월) 베타 시작

### 2. 왜 Que인가 — 지금 방식 vs Que (Before/After)
- **Before**: 보고는 회의로, 결정은 슬랙 스레드에 흘러가고, 일정은 머릿속에, 상태는 물어봐야. → 회사가 "사람 기억력"으로 돌아간다.
- **After**: 보고는 AI가 만들고, 결정은 기록되고, 병목은 스스로 손을 든다. 사람은 **결정만** 한다.
- 카피: **"일하는 시간을 돌려드립니다. 보고는 30초, 나머지는 AI가."**

### 3. 전체 지도 — 한 장으로 보는 Que
- 메뉴 4묶음(실행/운영/분석/바로가기) 다이어그램 + "어디서든 여는 법" 4개(사이드바·⌘K·폰 탭바·Slack 딥링크)

## 3부 — 툴별 (왜 → 어떻게 → 효과)

> 각 장 레이아웃: 왼쪽 3단 텍스트(왜/어떻게/효과) + 오른쪽 영상(GIF/mp4) 또는 스크린샷.

### 7. 데일리 · 스탠드업 체크인 ✅ (video-a-checkin.gif)
- **왜**: 아침 보고 회의는 8명×15분을 태운다. 보고를 위한 보고가 일을 밀어낸다.
- **어떻게**: 10시 Slack 버튼 → 폼 또는 AI 대화로 체크인. 어제·오늘 작업은 자동으로 붙고, 쓰는 건 **포커스 한마디**뿐. 막히면 막힘 칸에.
- **효과**: 회의 없는 아침. 막힘이 오전에 드러나 도움이 **당일** 간다. 지각 제출도 이력이 남는다.
- 카피: "보고는 30초. 나머지는 AI가."

### 8. 데일리 · OKR 탭 🎬 (30초: Objective 펼침→KR 진척 입력→작업 연결)
- **왜**: 오늘 한 일이 분기 목표와 무슨 상관인지 모르면 일이 소모가 된다. **Task ≠ Key Result.**
- **어떻게**: 분기 Objective 아래 월 KR(수동/작업연동/상태형 3방식). 작업 상세에서 KR에 연결.
- **효과**: "이 일이 어느 목표를 미는가"가 매일 보인다. 분기말 몰아치기 평가가 사라진다.

### 9. Que Copilot ✅ (video-b-copilot.gif)
- **왜**: 화면을 찾아다니고 폼을 채우는 시간. **물어보면 되는데.**
- **어떻게**: ⌘K 또는 Copilot 메뉴 → 조회("오늘 팀 막힌 거?")는 실데이터로만 답하고, 등록("내일 10시 ○○ 잡아줘")은 **확인 카드 → [실행]**. 프로젝트 생성도 대화로.
- **효과**: 등록 10초. 신입은 물어보며 배운다. 답은 실데이터만·변경은 확인 후만 — **환각·오실행 이중 방어**.
- 카피: "실행은 언제나 사람이 확인한 후에."

### 10. 작업 목록(오늘) 🎬 (30초: 완료 체크→자연어 등록→확인 카드)
- **왜**: 내 하루의 진입점이 없으면 오늘 할 일이 머릿속·포스트잇·슬랙에 흩어진다.
- **어떻게**: 아침에 열어 오늘 목록 확인 → 끝나면 체크 → 문제·홀드는 **사유·다음 액션과 함께** → 새 일은 자연어로("금요일까지 배너 시안").
- **효과**: "오늘 뭐부터?"가 사라진다. "그거 어떻게 됐어?" 메시지도 사라진다.

### 11. 프로젝트 · 칸반/간트 🎬 (40초: 칸반 드래그→실행 취소→간트 전환)
- **왜**: 프로젝트 전황판이 없으면 PM의 머릿속이 단일 장애점이 된다.
- **어떻게**: 4열 칸반 드래그(홀드·문제 열은 사유 필수), 실수하면 토스트 [실행 취소]. 보드/목록/캘린더/간트 전환. 카드 클릭=상세 드로어(댓글·도움 요청).
- **효과**: 어느 열에 카드가 쌓였는지 **5초**. 상태 공유 회의가 드래그 하나로 대체된다.

### 12. 일정 🎬 (20초: 주간 뷰→마일스톤 칩 클릭)
- **왜**: 회사 일정과 내 작업이 다른 곳에 살면, 충돌은 겹치고 나서야 안다.
- **어떻게**: 주/월 캘린더에 회사 일정+내 작업+마일스톤 칩(중요=옐로→오렌지 그라데이션). 충돌은 제안 카드로.
- **효과**: 일정 충돌이 **사전에** 드러난다. 외부 일정은 읽기 전용이라 실수로 못 건드린다.

### 13. 팀 현황 · Now 🎬 (20초: 운영 보드 훑기→상태 시트)
- **왜**: "지금 누가 뭐에 막혀 있나"를 알려면 8명에게 하나씩 물어야 했다.
- **어떻게**: 운영 보드에서 전원 상태·부하 확인. 타인 작업은 수정 대신 댓글·도움 요청·상태 확인 요청. 관리자는 주간/월간 리포트.
- **효과**: 병목이 스스로 손을 드는 팀. 점수화 없음 — **감시가 아니라 조망**.

### 14. 회의록 · Action ✅(회의록 초안 장면은 video-c-gantt.gif 후반) + 🎬(업로드→추출 20초)
- **왜**: 회의 결정이 채팅에 흩어지면 "그거 누가 하기로 했지?"가 매주 반복된다.
- **어떻게**: 회의록 업로드 → Action 자동 추출 → 담당·기한 있으면 Task 자동 생성, 없으면 **확인 필요**로 남아 사이드바 뱃지.
- **효과**: **기록되지 않은 결정 0건.** 회의가 실제 작업으로 이어진다.
- 카피: "결정이 곧 기록이다."

### 15. 반복·마일스톤 ✅ (video-c-gantt.gif)
- **왜**: 매주 같은 일을 매번 등록하는 낭비. 큰 기한이 위험해지는 걸 늦게 아는 사고.
- **어떻게**: 반복 템플릿은 한 번 켜두면 회차 자동 생성. 마일스톤은 기한 등록 + 최종 런칭 같은 건 **중요 체크**(옐로→오렌지 칩). 위험해지면 긴급 결정 카드(유지/연기/보류).
- **효과**: 기한 약속이 전 화면에 시각화되고, 위험이 **회의를 기다리지 않는다**.

### 16. 성과 🎬 (20초: 히트맵→기간 전환)
- **왜**: 감으로는 업무 몰림이 안 보인다. 몰림을 못 보면 재배분도 못 한다.
- **어떻게**: KPI·완료율·히트맵·프로젝트 진행률. 기간/월 선택.
- **효과**: 재배분의 **근거**가 생긴다. 개인 평가 아님 — 순위·점수 없음.

### 17. 결제요청 🎬 (20초: 요청 등록→상태 변경)
- **왜**: 결제 요청이 슬랙 DM으로 오가면 흘러가 누락되고, 됐는지 안 됐는지 아무도 모른다.
- **어떻게**: 요청 등록 → 대기/완료 추적. 계좌·금액은 권한별 마스킹.
- **효과**: "입금됐나요?" 문의가 사라진다.

### 18. 알림 · Slack 연동 🎬 (20초: DM 브리핑→딥링크 진입)
- **왜**: 도구를 열어야만 아는 정보는 이미 늦다.
- **어떻게**: 아침 개인 브리핑 DM·스탠드업 버튼·긴급 결정 카드가 Slack으로. 버튼 누르면 해당 화면 바로.
- **효과**: **Que를 안 열어도 필요한 것이 온다.** 폰 하단 탭으로 어디서든.

## 영상화 구상 (PPT와 대본 공유)

이 문서의 각 장 "왜/어떻게/효과"가 **그대로 내레이션 대본**이 되도록 썼습니다. 매체만 고르면 됩니다.

### 형태 3안
| 안 | 형태 | 분량 | 용도 | 추천 |
|---|---|---|---|---|
| **A. 툴별 쇼츠 시리즈** | 툴당 1편(30~60초) × 13편(6장 '변경·위기' 포함) | 편당 30~60초 | Slack에 하나씩 배포 — "오늘의 Que 한 가지". 베타 첫 2주 온보딩 리듬과 맞음 | ⭐권장 |
| B. 통합 온보딩 영상 | 이 목차 순서 그대로 1편 | 10~14분 | 신규 입사자 온보딩·아카이브 | 2순위(쇼츠 완성 후 이어붙이면 됨) |
| C. 발표 녹화 | PPT 발표를 화면+음성 녹화 | 25분 | 발표 불참자 공유용 | 발표 당일 부산물로 |

**A→B 순서를 권장**: 쇼츠 12편을 만들면 B는 인트로+이어붙이기로 거의 공짜. 각 편 구조(15초 왜 → 25초 어떻게 화면 → 10초 효과 자막)를 고정하면 템플릿화됩니다.

### 제작 파이프라인
1. **대본**: 이 문서 각 장을 그대로 낭독(장당 3~4문장, ~40초).
2. **화면**: ✅ 확보 3편(체크인·Copilot·간트) + 🎬 추가 촬영 9편 — Screen Studio(커서 확대·줌 자동, 발표 품질) 권장. 제가 Playwright 조작 GIF로 초벌을 떠드릴 수도 있음(무료·즉시, 커서 모션은 없음).
3. **자막·컷**: CapCut(무료). 각 편 끝에 카피 한 줄 고정 프레임.
4. **내레이션**: 직접 녹음 또는 TTS. 내레이션 없이 자막만으로도 쇼츠는 성립.
5. **인트로 2초**: 로고+툴 이름(Remotion 또는 CapCut 템플릿 — 12편 공용).
