## 3. 실행·배포 메모 (토요일 빠른 시작용)

### Que (`~/Desktop/que`, pnpm 모노레포)
```bash
pnpm install
pnpm dev                     # mock(인메모리) — 키 불필요, 포트 3000
# 실 DB로: (키는 data/.env, gitignore)
QUE_DB=supabase SUPABASE_URL=… SUPABASE_SECRET_KEY=… pnpm dev
pnpm -r typecheck && pnpm --filter @que/web lint && pnpm --filter @que/core test
```
- ⚠️ **dev 서버 떠 있는 동안 `pnpm build` 금지**(같은 `.next` 캐시 깨짐) — build 전 dev 종료
- 배포: `vercel --prod`(GRIFF 팀 `que`) · mock 인증 쿠키 `que-user=hwang-sunghyeon`(관리자)

### DayBlocks (`~/Desktop/todo_griff`, **npm**)
```bash
npm install
npm run dev                  # Vite, 포트 5173
npm run typecheck            # tsc -b --noEmit (lint 스크립트 없음, tsc가 게이트)
npm run build                # tsc + vite build
vercel --prod --yes          # GRIFF 팀 todo_griff → todo.griff.co.kr 별칭
```
- ⚠️ **pnpm 쓰지 말 것**(레포는 `package-lock.json`=npm) · 규범 스펙 = `DESIGN.md`(충돌 시 문서 우선)
- 테마 강제: 콘솔 `localStorage.theme='dark'|'light'` · 로그인 우회: "나중에"

### 공통 마무리 루틴
1. dev 서버 반드시 종료 → build/검증 → 2. 커밋(한국어 메시지 + Co-Authored-By) → 3. push → 4. `vercel --prod`
- 의미 있는 결정은 **`HANDOFF.md`**(Que) / **`DESIGN.md`**(DayBlocks) 갱신
- 릴리즈 전 **glados 게이트** 권장(이번 세션 내내 사용)
