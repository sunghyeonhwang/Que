# Que 홈(/home) 화면 명세 — 대표 · 관리자 · 사원

> 작성 2026-07-10. **C-3b 홈 정식 디자인(Figma)의 기초 명세.** 현행 구현(2026-07-10 라이브)을 정본으로 역할별 정보 구조·데이터·인터랙션을 정리했다. 디자인 시안은 이 명세의 **정보 구조와 우선순위를 유지**한 채 시각 표현만 다듬는 것을 권장한다(카드 추가·삭제·순서 변경은 명세 개정 후).
>
> 코드 기준: `apps/web/src/app/(app)/home/page.tsx` · `apps/web/src/lib/home-grade-data.ts` · `apps/web/src/components/home/{staff,manager,ceo}-home.tsx`

## 0. 설계 원칙

- **홈 = "오늘 무엇을 봐야 하는가"의 역할별 답.** 감시 도구가 아니라 병목·충돌·요청을 빨리 드러내는 운영 도구다(CLAUDE.md 톤).
- **역할(직급, grade)별 3분기 — 한 화면을 셋이 다르게 본다.**
  - `사원(staff)` = "**내 하루**": 내 일·내 일정·내게 온 요청이 전부. 팀 지표 없음.
  - `관리자(manager)` = "**어디가 막혔나**": 팀 병목·충돌·부하가 먼저, 본인 것은 하단 축소.
  - `대표(ceo)` = "**전사 조망**": 전사 KPI·프로젝트·클라이언트·전원 부하, 본인 것은 맨 아래.
- **개인 평가 아님.** 부하·히트맵·성과 지표는 업무 배분·병목 조정용이라는 성격을 화면에서 유지한다(성과 화면과 동일 원칙).
- **역할 판정은 서버가 세션 사용자로만 한다.** URL·클라이언트 입력으로 상위 화면을 열 수 없다(§5 권한).

### 역할(grade) 판정 규칙

| grade | 판정(rank 필드) | 홈 제목 | 부제 |
|---|---|---|---|
| `ceo` 대표 | rank = "대표" | **전사 조망** | 전사 진척·프로젝트·클라이언트 현황을 한눈에. |
| `manager` 관리자 | rank = "관리" | **어디가 막혔나** | 오늘 팀의 병목·충돌·부하를 먼저 봅니다. |
| `staff` 사원 | 그 외(누락 포함 폴백) | **어서오세요, {이름}님!** | 오늘 내 일과 내게 온 요청을 확인하세요. |

- 판정 함수: core `gradeForUser(user.rank)` — rank 문자열을 화면·데이터 계층에서 직접 비교하지 않는다.
- 쓰기 권한 축(role: admin/member)과 **별개 축**이다. grade는 "무엇이 보이는가", role은 "무엇을 고칠 수 있는가".

## 1. 공통 프레임 (세 역할 동일)

| 요소 | 내용 |
|---|---|
| 헤더 좌측 | 역할별 제목 + 부제(위 표) + **클라이언트 필터 배지**(전역 클라이언트 스위처 적용 중이면 "○○ 기준" 표시) |
| 헤더 우측 | 팀원 아바타 스택(+N 오버플로) + **작업 추가** 버튼(자연어 모달 — 해석→확인 카드→등록) |
| 우하단 | **FAB(플로팅 + 버튼)** — 상단 작업 추가와 같은 자연어 모달. 모바일·태블릿에서 주 진입점 |
| URL 상태 | `?hm=1..12` 히트맵 기준 월(staff·ceo만 사용). 잘못된 값은 현재월 폴백 |
| 데이터 스코프 | 전역 클라이언트 필터(쿠키)를 존중 — KPI·목록이 해당 클라이언트 기준으로 좁혀진다 |

**공통 카드 규격(HomeCard)**: 제목(좌) + 메타 수치/액션(우) + 본문. 장식·히어로 없음. 내부 스크롤 금지(홈은 세로 나열, 표·히트맵만 자체 가로 스크롤 허용).

## 2. 사원(staff) 홈 — "내 하루"

**한 줄 정의**: 출근해서 이 화면 하나로 "오늘 뭘 하지 + 나한테 뭐가 와 있지"가 끝난다. 팀 지표·타인 데이터는 없다.

**섹션 순서(위→아래)**:

| # | 섹션 | 구성 | 데이터·규칙 |
|---|---|---|---|
| 1 | **내 KPI 4장** | 총 작업 · 진행 중 · 완료 · 기한 초과 (아이콘+값+증감 배지) | **본인 작업만** 집계. 증감은 직전 동일 기간 대비 %, 극성 반영(기한초과는 줄면 green) |
| 2 | **오늘 할 일** (좌 2/5) | 체크리스트: 제목·상태 라벨·프로젝트 칩·마감 라벨·지연(red) 표시 | 본인 배정 오늘 작업. 완료 토글 즉시 커밋 |
| 2 | **개인 일정** (우 3/5) | 타임라인: 시간 라벨 + 색점(작업=상태색, 미팅=중립) | 본인 오늘 작업+일정. 우상단 확대 아이콘 → `/schedule` |
| 3 | **내게 온 요청** | 알림 인박스: 병목·확인필요·기한초과·결제 신호 행 + 회의록 요약 | **상단바 벨과 같은 소스**(getAlerts — 본인 대상 신호만). 행 클릭 → 해당 화면 딥링크 |
| 4 | **내 기여 히트맵** (좌 1/2) | 월 그리드 1행(본인) + "N월에 M개 기여" 캡션 + 월 선택(`?hm=`) | 개인 평가 아님 — 본인 기여 시각화 |
| 4 | **내 작업 성과** (우 1/2) | 라인 차트(완료·신규·기한초과) + 기간 라벨 | 본인 스코프 추이 |

**비어 있을 때**: 할 일 0건 → "오늘 배정된 작업이 없습니다"류 안내(빈 칸 금지). 요청 0건 → "지금 확인할 요청이 없습니다".

## 3. 관리자(manager) 홈 — "어디가 막혔나"

**한 줄 정의**: 아침에 열면 "오늘 누가 어디서 막혔고, 어떤 일정이 겹치고, 누가 과부하인가"를 30초 안에 파악한다. 본인 업무는 하단에 축소.

**섹션 순서(위→아래)**:

| # | 섹션 | 구성 | 데이터·규칙 |
|---|---|---|---|
| 1 | **팀 요약칩 5장** | 진행중 · **문제발생**(>0이면 red 숫자) · 홀드 · 마감 임박 · 응답 대기 | 팀 전체 오늘 기준 집계 |
| 2 | **주의 필요 — 어디가 막혔나** (좌 1/2) | 병목 목록: 유형(문제/홀드/응답대기/도움요청)·작업명·담당자·사유·재확인 시각·도움 대상 | **대표 담당 작업의 문제/홀드도 포함**(병목은 직급 무관 공유) |
| 2 | **일정 충돌** (우 1/2) | "○○ · HH:mm부터 A와 B가 겹칩니다" 문장 목록 | 오늘 일정 겹침 감지. 0건이면 "오늘 일정 충돌이 없습니다" |
| 3 | **팀 부하 — 업무 배분 조정용** | 팀원별 부하 막대 | **대표(ceo) 제외** — 배분 조정 대상만 |
| 4 | **확인 필요 Action** (좌 1/2) | 요약 배너: N건 + 회의록·후보 수 → 클릭 시 `/action` | 회의록 파생 Action 중 담당/마감 미정 건 |
| 4 | **결제 대기** (우 1/2) | 요약 배너: N건 + 마감 초과 있으면 red 강조 → 클릭 시 `/payments` | 초과 0이면 amber, >0이면 red 아이콘 |
| 5 | **내 오늘 할 일 · 내 개인 일정** (1/2 + 1/2) | 사원 홈 2번과 동일 컴포넌트, 축소 배치 | 관리자 본인 것 |

**비어 있을 때**: 병목 0건이 이 화면의 "좋은 상태" — "지금 막힌 작업이 없습니다"를 긍정 톤으로.

## 4. 대표(ceo) 홈 — "전사 조망"

**한 줄 정의**: 회사 전체가 어디로 가고 있는지(진척·위험·클라이언트·부하)를 한 화면에서 훑고, 본인 신호·일정은 맨 아래에서 확인한다.

**섹션 순서(위→아래)**:

| # | 섹션 | 구성 | 데이터·규칙 |
|---|---|---|---|
| 1 | **전사 KPI 4장** | 총 작업 · 진행 중 · 완료 · 기한 초과 (증감 배지 포함) | **전사** 집계(클라이언트 필터 적용 시 그 스코프) |
| 2 | **완료 추이** (좌 1/2) | 월별 완료 막대 차트 + 기간 라벨 | 성과 화면 컴포넌트 재사용 |
| 2 | **프로젝트 진행률** (우 1/2) | 전체 진행률 %(우상단) + 프로젝트별 막대 목록 | 활성 프로젝트 |
| 3 | **위험 마일스톤** (좌 1/2) | riskStatus ≠ on_track 목록(주의=amber, 지연=red) | 마일스톤 위험만 — 정상은 안 보여줌 |
| 3 | **현재 막힘** (우 1/2) | 문제/홀드 작업 목록(담당자·사유) | 전사 병목 |
| 4 | **클라이언트별 현황** | 클라이언트 행: 활성 프로젝트 수 · 평균 진행률 · 막힘 작업 · 열린 작업 | 거래처 관점 요약(대표 전용 카드) |
| 5 | **전 인원 부하** (좌 1/2) | 전원 부하 막대(**대표 포함 전 인원**) | 관리자 홈과 달리 전원 |
| 5 | **전 인원 히트맵** (우 1/2) | 팀원×날짜 월 그리드 + 월 선택(`?hm=`) | 성과 화면 히트맵 재사용 |
| 6 | **결제 대기** | 요약 배너(관리자 홈과 동일) → `/payments` | 마감 초과 red 강조 |
| 7 | **내게 온 요청** | 알림 인박스(사원 홈 3번과 동일 — 본인 신호) | 대표 본인 대상 신호 |
| 8 | **내 오늘 할 일 · 내 개인 일정** (1/2 + 1/2) | 본인 축소 배치 | 대표 본인 것 |

## 5. 데이터 스코프 · 권한 방어 (3중)

1. **분기**: grade는 `getGradeHomeData(user, …)` 내부에서 세션 user로만 판정 — URL·파라미터로 상위 홈 요청 불가.
2. **데이터 게이트**: 관리자/대표 전용 번들(리포트·전원 부하 등)은 `getAdminReportData`(role 게이트)·`getPerformanceData`(viewer 재유도)를 통과해야 채워진다 — 사원이 함수를 직접 불러도 빈 골격.
3. **컴포넌트**: staff 홈에는 타인 데이터 prop 자체가 없다(타입 수준 분리 — `StaffHomeData | ManagerHomeData | CeoHomeData` 유니온).

기타: 비공개 자리비움·PII는 홈에 노출하지 않는다. 알림(내게 온 요청)은 상단바 벨·`/notifications`와 같은 소스라 수치가 항상 일치해야 한다.

## 6. 반응형 · 접근성 (검수 4해상도)

| 해상도 | 규칙 |
|---|---|
| FHD 1920 / 1366 | 위 표의 그리드(1/2, 2/5+3/5, KPI 4열) 그대로 |
| 태블릿 가로 1024 | KPI 2×2, 2단 카드는 1단으로 순차 적층(xl 브레이크포인트 미만) |
| 태블릿 세로 768 | 전부 1열 세로 나열. 사이드바는 접힘(레일). FAB가 주 작업 추가 진입점 |

- 터치 대상 40px+(권장 44px): 완료 토글·요청 행·배너·월 선택 모두 행 단위 터치.
- 색상 단독 금지: 지연=red 텍스트+아이콘, 문제발생 칩=숫자 색+라벨, 히트맵=값 숫자 병기.
- 히트맵·표는 카드 내부 가로 스크롤 허용, 페이지 가로 스크롤 금지.

## 7. 디자인 가이드 (Figma 작업 시)

- **토큰 준수**: shadcn 글로벌 theme token을 브랜드 색으로 덮어쓰지 않는다. 색 의미 고정 — green=진행/완료, blue=예정/정보, amber=주의/대기, red=문제/취소, violet=회의록/응답대기.
- **운영 도구 톤**: 히어로·일러스트·카드 중첩 금지. 숫자·상태·액션이 먼저 보이게. 카드 안 카드 금지(요약 배너는 단일 행 카드).
- **다크 모드**: 전 카드 라이트/다크 동시 시안(현행 토큰이 양 모드 지원).
- **재사용 컴포넌트**: KPI 카드·히트맵·라인/막대 차트·진행률 목록은 성과 화면과 공유 — 홈에서만 다르게 그리지 않는다(한 벌 유지).
- **변경 절차**: 카드 추가/삭제/순서 변경이 필요한 시안은 이 명세를 먼저 개정(사용자 승인)한 뒤 반영한다.

## 8. 알려진 개선 후보 (시안에서 해결하면 좋은 것)

- 사원 홈 KPI 4장이 첫 화면을 차지하는데, 사원에게 가장 급한 건 "오늘 할 일"이다 — 시안에서 KPI를 압축(1행 스트립)하고 할 일을 끌어올리는 안 검토 가치.
- 관리자 홈 "일정 충돌"이 문장 나열이라 3건 이상이면 스캔이 어렵다 — 표 형태 검토.
- 대표 홈이 8섹션 세로 나열로 길다 — 시안에서 2단 조망(좌: 지표, 우: 목록) 등 밀도 개선 여지.
- "내게 온 요청"과 상단바 벨의 중복 노출 — 홈에서는 요약 수치+상위 3건으로 줄이는 안.
