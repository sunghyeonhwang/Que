# Que 베타 QA 확인 시트

베타 시작(7/20) 전에 팀 전원이 직접 따라 하며 확인하는 체크리스트입니다. 인쇄하거나 화면에 띄워 놓고 위에서부터 순서대로 진행하세요.

## 시작하기 전에

- **발견한 문제는 왼쪽 메뉴 [수정사항](/revisions)에 기록하세요.** 메뉴(어느 화면) · 위치(화면 안 어디) · 오류사항(무엇이 이상한지)을 적으면 됩니다. 관리자에게 따로 알릴 필요 없이 팀 전체가 함께 확인합니다.
- 각 항목은 **"무엇을 하고 → 무엇이 보여야 정상"** 형식입니다. 화살표(→) 뒤의 결과가 그대로 보이면 체크하고, 다르면 수정사항에 기록한 뒤 다음 항목으로 넘어가세요.
- 데이터를 실제로 만들고 바꾸는 테스트입니다. 테스트로 만든 작업·결제·수정사항 제목 앞에는 `[QA]`를 붙여 나중에 정리하기 쉽게 하세요.
- 시트는 3부입니다. **1부는 전원**, **2부는 관리자·PM만 추가로**, **3부는 대표만 추가로** 진행합니다.

---

# 1부. 전원 공통 (팀원 기준)

**준비물**: 본인 계정(회사 이메일 + 비밀번호), 데스크톱 또는 노트북 브라우저, 스마트폰, 회사 Slack 로그인 상태
**예상 소요 시간**: 약 90분 (섹션을 나눠 여러 번에 걸쳐 해도 됩니다)

## 1-1. 로그인

- [ ] 로그인 화면에서 회사 이메일과 비밀번호를 입력하고 로그인한다 → **홈** 화면이 열린다 (모든 직급 공통).
- [ ] 비밀번호를 일부러 틀리게 입력하고 로그인한다 → "이메일 또는 비밀번호가 올바르지 않습니다." 빨간 안내가 표시되고 로그인되지 않는다.
- [ ] 비밀번호 칸 오른쪽 눈 모양 버튼을 누른다 → 가려진 비밀번호가 보이고, 다시 누르면 가려진다.
- [ ] **아이디 기억하기**에 체크하고 로그인한 뒤, 로그아웃하고 로그인 화면으로 돌아온다 → 이메일 칸이 자동으로 채워져 있고, **비밀번호 칸은 비어 있다**.
- [ ] 오른쪽 위 내 계정(이름 첫 글자 아이콘)을 누르고 **로그아웃**을 누른다 → 로그인 화면으로 돌아온다.

## 1-2. 화면 구성 · 검색 · 단축키

- [ ] 왼쪽 메뉴를 위에서 아래로 훑는다 → **메뉴**(홈~멤버) · **바로가기**(간트·뷰·투두·컬러) · **기타**(결제요청~설정) 세 묶음이 보인다. 일반 팀원에게는 **클라이언트** 메뉴가 보이지 않는 것이 정상이다.
- [ ] 현재 화면의 메뉴 항목을 확인한다 → 진하게(활성) 표시된다.
- [ ] 바로가기 묶음의 **간트·뷰·투두·컬러**를 각각 누른다 → 각 항목이 **새 탭**으로 열리고, Que 화면은 그대로 유지된다.
- [ ] 위쪽 가운데 검색창에 작업 제목의 일부를 입력한다 → 결과 목록이 나오고, 항목을 누르면 해당 화면으로 이동한다.
- [ ] 아무 화면에서나 **⌘K**(윈도우 Ctrl+K)를 누른다 → 가운데에 빠른 이동·검색 창이 열리고, 화면 이름을 입력해 Enter를 누르면 그 화면으로 이동한다.
- [ ] 입력 칸 밖에서 **`?`** 키를 누른다 → 단축키 도움말 창이 열린다.

## 1-3. 알림 (종 모양 · 알림 페이지)

- [ ] 오른쪽 위 종 모양 버튼을 누른다 → 내게 관련된 알림 목록이 열린다. 읽지 않은 알림이 있으면 종 위에 빨간 숫자가 보인다.
- [ ] 알림 항목 하나를 누른다 → 해당 화면으로 이동하고, 그 알림이 읽음으로 바뀐다(뱃지 수 감소).
- [ ] 알림 목록 아래 **모두 보기**를 누른다 → 알림 전체 페이지가 열리고, **모두 읽음** 버튼으로 한 번에 정리된다.
- [ ] 홈의 **우선 확인** 개수와 종 알림 개수를 비교한다 → 두 곳의 개수가 일치한다.

## 1-4. 홈

- [ ] 왼쪽 메뉴에서 **홈**을 누른다 → 직급에 맞는 제목이 보인다 (사원=내 하루, 관리자=팀 운영, 대표=전사 현황).
- [ ] 핵심 현황 숫자 카드 하나를 누른다 → 관련 화면(작업 목록·프로젝트 등)으로 이동한다.
- [ ] 오늘 요약의 **AI 브리핑 생성** 버튼을 누른다 → 잠시 후 요약 문장이 생성되고, 아래에 외부(Google) 전송 안내가 표시된다.
- [ ] **오늘 할 일**에서 내 작업의 원형 버튼을 누른다 → 완료로 표시된다.
- [ ] 목록이 긴 칸을 스크롤한다 → 그 칸 안에서만 스크롤되고 화면 전체는 흔들리지 않는다.

## 1-5. 데일리 (스탠드업 체크인)

- [ ] 왼쪽 메뉴에서 **데일리**를 누른다 → 맨 위에 내 체크인 폼, 아래에 팀 전원 카드 그리드와 제출 진행률(N/8)이 보인다.
- [ ] **오늘의 포커스**를 비운 채 제출을 시도한다 → 제출되지 않는다 (포커스는 필수).
- [ ] **AI 초안 받기**를 누른다 → 포커스·부연·막힘 초안이 폼에 채워지기만 하고, **자동으로 제출되지 않는다**.
- [ ] 포커스를 한마디 적고 **체크인 제출**을 누른다 → 폼이 "제출됨"으로 접히고, 내 카드가 팀 그리드에 나타난다.
- [ ] 제출한 체크인의 **수정** 버튼을 누른다 → 다시 열어 고칠 수 있다 (본인 체크인만).
- [ ] **대화로 체크인**을 눌러 AI와 짧게 문답한다 → 마지막에 정리된 내용을 확인하고 제출할 수 있고, "폼으로 입력"으로 되돌아올 수 있다.
- [ ] 막힘이 있는 팀원 카드를 확인한다 → 빨간 **막힘** 표시로 강조된다.
- [ ] 화면 맨 아래 **AI 팀 요약** 영역을 확인한다 → 전원 제출 전에는 "곧 생성됩니다"와 현재 제출 인원이, 생성 후에는 막힘·흐름·추천 액션 요약이 보인다.

## 1-6. 데일리 — OKR 탭

- [ ] 데일리 화면 위쪽 **OKR** 탭을 누른다 (또는 사이드바 데일리 하위 OKR) → 분기 목표(Objective) 카드 목록이 보인다.
- [ ] 목표 카드를 누른다 → 펼쳐지면서 핵심결과(KR)마다 진척 바·측정 방식·소유자·연결 작업 수가 보인다.
- [ ] 위쪽 분기 선택을 바꾼다 → 선택한 분기의 목표만 표시된다.
- [ ] **내가 소유자가 아닌** 수동 지표 KR의 진척 입력을 시도한다 → 입력할 수 없다 (진척 입력은 소유자·관리자만).
- [ ] 작업 목록이나 프로젝트에서 **내 작업** 상세를 열고 **핵심결과** 칸에서 KR을 연결한다 → OKR 탭에서 그 KR의 연결 작업 수가 늘어난다.

## 1-7. Copilot (AI 대화)

- [ ] 왼쪽 메뉴에서 **Copilot**을 누른다 → 화면 전체를 채우는 채팅 화면이 열리고, 가운데에 예시 질문 칩이 보인다.
- [ ] "오늘 나 뭐 해야 해?"라고 묻는다 → 실제 내 작업이 정리되어 답하고, 답변 아래 출처 칩을 누르면 해당 화면으로 이동한다.
- [ ] "내일 마감으로 [QA] 테스트 작업 만들어줘"라고 요청한다 → 바로 실행되지 않고 **확인 카드**(제목·담당·기한 요약)가 표시된다. **[실행]을 누르기 전에는 작업 목록에 생기지 않는다.**
- [ ] 확인 카드에서 **실행**을 누른다 → 작업이 실제로 생성되고 작업 목록에서 확인된다.
- [ ] 다른 쓰기 요청을 하고 확인 카드에서 **취소**를 누른다 → 아무것도 반영되지 않는다.
- [ ] **타인 작업**의 상태 변경을 요청하고 실행을 누른다 → 실행이 거부되고 카드 아래에 그 이유가 표시된다.
- [ ] "새 프로젝트 만들어줘"라고 요청한다 → 확인 카드가 나오고 실행하면 프로젝트가 생성된다 (프로젝트 생성은 Copilot으로 누구나 가능).
- [ ] **⌘K** 창에 질문을 입력하고 맨 아래 **✨ AI에게 묻기**를 누른다 → 창이 채팅으로 확장되고, 채팅 위쪽 **전체 화면**을 누르면 Copilot 전용 화면으로 넘어간다 (대화는 새로 시작).

## 1-8. 작업 목록 (오늘)

- [ ] 왼쪽 메뉴에서 **작업 목록**을 누른다 → 맨 위에 오늘 요약 한 줄, 아래에 내 작업 목록이 보인다.
- [ ] **모든 작업 · 오늘 · 예정 · 완료됨** 필터를 차례로 누른다 → 목록이 해당 기준으로 좁혀지고 버튼 옆 숫자와 개수가 맞는다.
- [ ] 내 작업 행 오른쪽 끝 원형 체크 버튼을 누른다 → **완료**로 바뀌고 폭죽(컨페티) 효과가 나온다.
- [ ] 같은 버튼을 다시 누른다 → **진행중**으로 되돌아온다.
- [ ] 내 작업 제목을 눌러 상세 창을 열고 상태를 **홀드**로 바꾸되 **사유를 비워 둔다** → 변경 버튼이 눌리지 않거나 거부된다 (사유는 필수).
- [ ] 사유를 채우고 홀드로 변경한다 → 상태가 홀드(노랑)로 바뀐다. 다음 액션·도움 필요한 사람·다시 확인할 시간도 함께 입력할 수 있다.
- [ ] **타인 작업**의 상세를 연다 → 상태 버튼이 보이지 않고 "본인, 프로젝트 담당자, 관리자만 수정할 수 있습니다" 안내가 보이며, 댓글·도움 요청·상태 확인 요청만 가능하다.
- [ ] **새 작업 등록** 칸에 자연어로 입력한다 (예: "내일 오후 2시 [QA] 배너 시안 검토") → 등록 전에 내용을 정리한 **확인 카드**가 먼저 표시되고, 확인 후 등록(또는 ⌘Enter)해야 목록에 추가된다.
- [ ] 방금 만든 [QA] 작업을 상세에서 삭제(취소)한다 → 성공 토스트에 **[실행 취소]** 버튼이 붙고, 누르면 작업이 원래대로 돌아온다.
- [ ] **현황 상세**를 펼친다 → 내 타임라인 · 작업 상태 확인 · 하루 마감 · 주의 필요 칸이 나타나고, 접으면 목록이 넓어진다.
- [ ] 작업 상태 확인(자동 체크인) 카드가 있으면 응답한다 → 숫자키 1~7로도 선택되고, 응답하면 카드가 정리된다.

## 1-9. 프로젝트 (보드 · 목록 · 캘린더 · 간트)

- [ ] 왼쪽 메뉴에서 **프로젝트**를 누른다 → 보드(칸반)가 열리고 컬럼은 예정 · 진행중 · 홀드·문제 · 완료 4열이다.
- [ ] 보드/목록/캘린더/간트 뷰를 차례로 전환한다 → 각 뷰가 깨짐 없이 그려진다.
- [ ] **내 작업** 카드를 다른 열로 드래그한다 → 이동되고, 토스트에 **[실행 취소]**가 붙는다. 누르면 이전 상태로 복원된다.
- [ ] 내 작업 카드를 **홀드·문제** 열로 드래그한다 → 사유 입력이 먼저 요구되고, 사유 없이는 이동되지 않는다.
- [ ] **타인 작업** 카드의 이동·편집을 시도한다 → 반영되지 않고 권한 안내가 표시된다.
- [ ] 간트 뷰를 연다 → 오늘 위치로 자동 스크롤되고, 오늘이 점선과 칩으로 표시되며, 상단에 범례가 보인다.
- [ ] 간트에서 선행 작업이 연결된 작업을 확인한다 → 막대 사이에 화살표가 그려지고, 선행 마감이 지났는데 미완료면 노란 점선과 경고 아이콘·문장이 표시된다.
- [ ] 카드 상세 드로어의 **선행 작업**에서 검색해 연결/해제한다 (내 작업) → 즉시 반영되고 간트 화살표가 바뀐다.
- [ ] 캘린더·간트에서 **마일스톤 칩**을 확인한다 → 일반은 시안→옐로 그라데이션, **중요 마일스톤은 옐로→오렌지 그라데이션**으로 구분된다.
- [ ] 권한 없는 마일스톤 칩을 누른다 → 조회 전용 상세가 열리고 "프로젝트 담당자·관리자만 수정할 수 있습니다" 안내가 보인다.

## 1-10. 일정

- [ ] 왼쪽 메뉴에서 **일정**을 누른다 → 달력이 열리고 맨 위에 오늘 날짜와 남은 마감·미팅 안내가 보인다.
- [ ] 일간/주간/월간 보기를 전환한다 (버튼 또는 키보드 **D / W / M / 3**) → 각 보기가 정상 표시된다.
- [ ] **← →** 키와 **T** 키를 누른다 → 이전/다음 기간으로 이동하고 오늘로 돌아온다.
- [ ] **외부 회사 일정** 블록을 확인한다 → 자물쇠 표시가 있고, 상세를 열면 "읽기 전용 일정입니다"로 표시되며 수정·이동할 수 없다.
- [ ] 타인의 비공개 일정을 확인한다 → 제목 대신 **자리비움**으로만 표시된다.
- [ ] **새로 추가**로 작업 또는 일정(미팅)을 만든다 → 저장하면 달력에 표시된다.
- [ ] 마일스톤이 있는 날짜를 확인한다 → 그라데이션 마일스톤 칩이 표시되고, 누르면 상세가 열린다.
- [ ] 내 작업 블록의 상세에서 **날짜/시간 변경**으로 일정을 옮긴다 → 달력에서 새 자리로 이동한다.

## 1-11. 팀 현황

- [ ] 왼쪽 메뉴에서 **팀 현황**을 누른다 → 요약 카드 5개(진행중·문제발생·홀드·마감 임박·상태 응답 대기)와 운영 보드가 보인다.
- [ ] 사람별 오늘 시간표에서 시간이 겹치는 팀원을 확인한다 → 이름 옆에 **충돌 N** 표시가 보이고, 일정 충돌 칸에 문장으로 안내된다.
- [ ] 시간표에서 **타인 작업**을 누른다 → 상태 버튼 없이 권한 안내가 보이고 댓글만 가능하다.
- [ ] **스탠드업** 보기로 전환한다 → 팀원별 어제·오늘·막힘 카드가 보이고, 조회 전용이다.
- [ ] 일반 팀원 계정에서 보기 버튼을 확인한다 → **리포트** 버튼이 보이지 않는 것이 정상이다.

## 1-12. 회의록 · 확인필요 (Action)

- [ ] 왼쪽 메뉴에서 **회의록**을 누른다 → 상태 요약 카드 4개(회의록·추출 대기·Action 후보·확인 필요)가 보인다.
- [ ] 회의록 파일(.md/.txt)을 업로드한다 (공개 범위 선택 포함) → 목록에 **추출 대기**(노랑)로 추가되고, **원문** 버튼으로 원본을 열람할 수 있다.
- [ ] **Action 추출**을 누른다 → 잠시 후 **후보 N건**(파랑)으로 바뀐다.
- [ ] Action 화면에서 담당자·마감일이 **없는** 항목을 확인한다 → **확인 필요** 상태로 남아 있고, 자동으로 작업이 되지 않는다.
- [ ] 담당자와 마감일을 지정하고 **저장** → **Task 생성**을 누른다 → 작업이 생성되어 담당자의 작업 목록·일정에 표시된다.
- [ ] 항목을 **보류** 또는 **무시**로 처리한다 → 상태가 바뀌고 목록이 정리된다.
- [ ] 확인 필요 건을 처리한 뒤 사이드바 **회의록** 뱃지를 본다 → 숫자가 줄어든다.

## 1-13. Now (운영표)

- [ ] 왼쪽 메뉴에서 **Now**를 누른다 → 숫자 카드(오늘 캘린더 항목·Action 후보·문제/홀드·오늘 마감·담당자 확인 필요)와 표가 보인다.
- [ ] **전체 · 내 항목 · 문제** 필터를 누른다 → 표가 해당 기준으로 좁혀진다.
- [ ] 담당자 없는 항목을 확인한다 → 빨간 글씨로 **미지정**이 표시된다.
- [ ] **Action 확정하러 가기**를 누른다 → 회의록(Action) 화면으로 이동한다.

## 1-14. 반복·마일스톤

- [ ] 왼쪽 메뉴에서 **반복·마일스톤**을 누른다 → **반복 업무 · 마일스톤** 두 탭과 탭별 설명 블록이 보인다.
- [ ] 반복 업무 템플릿을 하나 등록한다 (이름·담당자·주기) → 목록에 추가되고 켜짐 상태로 표시된다.
- [ ] 템플릿을 껐다 켠다 → 상태 표시가 바뀌고, 꺼도 이미 생성된 회차 작업은 사라지지 않는다.
- [ ] 마일스톤 탭에서 **내가 담당하지 않는** 프로젝트의 마일스톤 수정을 시도한다 → 수정할 수 없다 (담당자·관리자만).
- [ ] 기한을 넘겨 끝난 마일스톤이 있으면 확인한다 → **회고 필요** 표시가 보인다 (회고 작성은 담당자·관리자만 — 2부에서 확인).

## 1-15. 성과

- [ ] 왼쪽 메뉴에서 **성과**를 누른다 → 맨 위에 "개인 평가가 아니라 업무 배분과 병목 조정용" 문구가 보인다. 사원 계정은 탭 없이 **내 성과**만 보인다.
- [ ] 숫자 카드 4개(총 작업·진행 중·완료·기한 초과)를 확인한다 → 숫자와 증감 화살표가 표시된다.
- [ ] 히트맵의 월 선택을 바꾼다 → 해당 달의 그리드로 다시 그려지고, 히트맵은 카드 안에서 스크롤된다.
- [ ] 기한 초과 추이의 기간(4~26주)을 바꾼다 → 그래프가 해당 기간으로 갱신된다.

## 1-16. 멤버

- [ ] 왼쪽 메뉴에서 **멤버**를 누른다 → 요약 카드 4개와 팀원 카드(이름·이메일·부서)가 보이고, 전화번호·생년월일 같은 개인정보는 없다.
- [ ] 팀원 카드의 **자세히**를 누른다 → 프로필·최근 활동·히트맵·작업 성과 그래프가 보인다.
- [ ] 화면 어디에도 팀원 추가·편집·삭제 버튼이 없는지 확인한다 → 조회 전용이 정상이다 (관리자에게만 오른쪽 위 **직원 관리** 버튼이 보인다).

## 1-17. 결제요청

- [ ] 왼쪽 메뉴에서 **결제요청**을 누른다 → 상태 카드 4개(대기·마감 초과·완료·취소)가 보인다.
- [ ] 제목·은행·계좌·금액·분류를 채워 `[QA]` 결제 요청을 등록한다 → 목록에 **대기**(노랑)로 추가된다. 필수값이 비면 등록 버튼이 눌리지 않는다.
- [ ] **타인이 올린** 요청의 계좌·금액을 확인한다 → 계좌는 뒷자리만(`•••• 1234`), 금액은 **비공개**로 가려져 있고 복사 버튼도 없다.
- [ ] **내가 올린** 요청을 확인한다 → 계좌·금액 원래 값과 복사 버튼이 보인다.
- [ ] 일반 팀원 계정에서 타인 요청의 완료(원형 체크)를 시도한다 → 처리할 수 없다 (입금 완료는 관리자만).
- [ ] 내가 올린 [QA] 요청을 **취소**한다 → 상태가 취소(빨강)로 바뀐다.

## 1-18. 수정사항

- [ ] 왼쪽 메뉴에서 **수정사항**을 누른다 → 등록 폼과 목록이 보인다.
- [ ] 메뉴·위치·오류사항을 채워 등록한다 → 목록에 **미해결**(빨강)로 추가되고 작성자·시간이 자동 기록된다.
- [ ] 아무 항목의 상태를 **보류** 또는 **해결**로 바꾼다 → 누구나 바꿀 수 있고 색이 바뀐다(노랑/초록).
- [ ] **전체 · 미해결 · 보류 · 해결** 필터를 누른다 → 목록이 좁혀진다.

## 1-19. 설정

- [ ] **설정 › 모양**에서 테마를 다크로 바꾼다 → 즉시 화면이 어두워지고, 라이트로 되돌리면 밝아진다. 밀도·글꼴 변경도 즉시 반영된다.
- [ ] **설정 › 보안**에서 비밀번호를 변경한다 → 완료 안내가 표시되고, 로그아웃 후 새 비밀번호로 로그인된다. (변경한 비밀번호는 잊지 마세요.)
- [ ] **설정 › 토큰**에서 라벨을 적고 토큰을 발급한다 → 토큰 값이 **그 순간 한 번만** 표시되고, 화면을 벗어나면 다시 볼 수 없다. **폐기**하면 목록에서 정리된다.
- [ ] **설정 › 가져오기**에서 샘플 CSV를 내려받고, 채운 파일로 **미리보기**를 한다 → 줄마다 가능/오류 판정이 표시되고, 미리보기 없이 바로 등록할 수 없다.

## 1-20. 도움말 · 설계 FAQ · MCP·CLI

- [ ] 왼쪽 메뉴에서 **도움말**을 누른다 → 왼쪽 섹션 목차와 본문이 보이고, 검색으로 원하는 절을 찾을 수 있다.
- [ ] 왼쪽 메뉴에서 **설계 FAQ**를 누른다 → "왜 이렇게 만들었나" 질문 목록(왜 남의 작업은 수정할 수 없나요 등)이 열린다.
- [ ] 왼쪽 메뉴에서 **MCP · CLI**를 누른다 → 사전 준비·CLI·MCP 온보딩 안내가 보이고, 명령 블록의 복사 버튼이 동작한다. 화면 어디에도 **실제 토큰 값은 노출되지 않는다**.

## 1-21. 클라이언트(거래처) 전환

- [ ] 메뉴 위(또는 좁은 화면에서는 상단바)의 거래처 전환 버튼에서 거래처 하나를 선택한다 → 작업 목록·일정·성과·팀 현황이 그 거래처 기준으로 좁혀지고 "○○ 기준" 배지가 보인다.
- [ ] **전체 클라이언트**로 되돌린다 → 전체 데이터가 다시 표시된다. (작업이 안 보일 때 가장 먼저 확인할 지점입니다.)

## 1-22. Slack 알림 수신

- [ ] 내 작업을 사유와 함께 **문제발생** 또는 **홀드**로 바꾼다 → 팀 Slack 채널에 작업명·담당자·사유와 **Que에서 열기** 링크가 올라오고, 링크를 누르면 해당 화면이 열린다. (확인 후 상태를 원래대로 되돌리세요.)
- [ ] 아침에 팀 채널을 확인한다 → 팀 스탠드업 요약이 올라와 있다. 오전 10시에는 스탠드업 오픈 알림과 개인 DM 안내가 온다.
- [ ] 밤 10시~아침 8시 사이에 생긴 팀 채널 알림이 있었다면 확인한다 → 밤에 오지 않고 아침에 몰아서 발송된다.
- 참고: 개인 DM 데일리 브리핑(오전 9:50)·할 일 배정 DM은 **단계 롤아웃 중**이라 현재 대표 계정만 수신할 수 있습니다(3부에서 확인). 베타 전 전원 확대되면 이 항목을 전원이 다시 확인하세요.

## 1-23. 모바일 (스마트폰, 화면 폭 768 미만)

- [ ] 폰 브라우저로 로그인한다 → 화면 아래에 하단 탭 5개(**홈 · 데일리 · Copilot · 알림 · 메뉴**)가 보인다.
- [ ] 하단 탭을 하나씩 눌러 다섯 곳을 모두 이동한다 → 각 화면이 열리고 현재 탭이 강조된다. 맨 오른쪽 **메뉴** 탭은 전체 메뉴 시트가 옆에서 열린다.
- [ ] 폰에서 **데일리** 탭으로 이동해 체크인을 제출한다 → 데스크톱과 동일하게 제출되고 팀 그리드에 반영된다.
- [ ] 폰에서 **Copilot** 탭으로 대화한다 → 입력창이 키보드 위로 올라오고 하단 탭과 겹치지 않으며, 대화가 정상 진행된다.
- [ ] 읽지 않은 알림이 있는 상태에서 **알림** 탭을 본다 → 탭에 미읽음 뱃지가 보이고, 상단바 종과 같은 숫자다.

# 2부. 관리자·PM 추가

1부를 먼저 마친 뒤 진행하세요. **관리자 권한 계정** 또는 **프로젝트 담당자(PM)** 대상입니다.

**준비물**: 관리자 권한 계정(또는 본인이 담당자인 프로젝트가 있는 계정), 비교용으로 일반 팀원 계정 1개(권한 차이 확인용)
**예상 소요 시간**: 약 50분

## 2-1. 클라이언트 관리 (관리자 전용)

- [ ] 관리자로 로그인한다 → 기타 묶음에 **클라이언트** 메뉴가 보인다. (일반 팀원 계정에는 보이지 않는 것도 함께 확인)
- [ ] `[QA]` 클라이언트를 추가한다 → 목록에 추가되고 거래처 전환 스위처에도 나타난다.
- [ ] 클라이언트 이름을 수정하고 보관한다 → 이름이 바뀌고, 보관하면 활성 목록·스위처에서 빠진다.
- [ ] 클라이언트 아래에 프로젝트를 추가하고, 다른 클라이언트로 재배정한다 → 소속이 바뀌어 표시된다.

## 2-2. 설정 › 직원 관리 (관리자 전용)

- [ ] **설정 › 직원 관리** 탭(또는 멤버 화면 오른쪽 위 **직원 관리** 버튼)을 연다 → 직원 추가 폼과 직원 목록 표가 보인다.
- [ ] 테스트 직원을 추가한다 (@griff.co.kr 이메일만 허용) → **임시 비밀번호가 그 순간 한 번만** 표시된다.
- [ ] 직원 정보(직급·부서)를 편집한다 → 목록에 반영된다 (이름은 편집 불가).
- [ ] 본인 권한 변경을 시도한다 → 변경할 수 없다. 마지막 활성 관리자 강등도 막힌다.
- [ ] 테스트 직원을 **비활성**한다 → 로그인·조회가 차단되고, **복구**로 되돌릴 수 있다. 열린 작업·켜진 반복 템플릿이 있으면 먼저 정리하라는 안내가 나온다.
- [ ] **비번 재설정**을 누른다 → 새 임시 비밀번호가 발급·표시된다.

## 2-3. 팀 현황 — 리포트 · AI 분석 (관리자 전용)

- [ ] 팀 현황에서 **리포트** 버튼을 누른다 → 관리자에게만 보이는 리포트 뷰가 열린다 (전체 현황·운영 건강도·진척·업무 흐름·막힌 작업·부하 분포).
- [ ] 주간/월간 기간을 전환한다 → 수치가 해당 기간으로 다시 계산된다.
- [ ] 전체 현황의 숫자 카드를 누른다 → 관련 화면으로 이동한다.
- [ ] 하단 **AI 분석** 카드에서 분석을 실행한다 → 잠시 후 병목·조치 제안이 생성되고, **참고 의견·개인 평가 아님·외부(Google) 전송** 고지가 함께 표시된다. (자동 실행이 아니라 버튼을 누를 때만 실행)

## 2-4. 긴급 결정 카드 (담당자·관리자)

- [ ] 위험 마일스톤이 있을 때 **데일리**를 연다 → 화면 위쪽에 빨간 **긴급 결정 대기** 카드(사유·진행률·마감)가 보인다. 6건 이상이면 "나머지 N건 보기"로 접힌다.
- [ ] **유지**를 누른다 → 기한이 그대로 확정되고 **카드가 사라진다**.
- [ ] 다른 카드에서 **연기**를 누르고 새 마감일을 입력해 확정한다 → 마감일이 바뀌고 카드가 사라진다.
- [ ] 다른 카드에서 **보류**를 누르되 사유를 비워 둔다 → 확정 버튼이 눌리지 않는다. 사유를 적으면 '주의' 상태로 반영되고 카드가 사라진다.
- [ ] 일반 팀원 계정으로 같은 카드를 본다 → 결정 버튼 대신 "결정 권한은 담당자·관리자에게 있습니다 (조회 전용)" 안내가 보인다.

## 2-5. 회의 진행 모드 (주간 통합 회의)

- [ ] 데일리 오른쪽 위 **회의 진행 모드**를 누른다 → 5섹션 스텝퍼(지난주 요약 → 이번 주 조망 → 마일스톤 안건 → 결정 필요 → 팀 라운드)가 큰 글씨로 열린다.
- [ ] 이전/다음으로 단계를 이동한다 → 주소의 step이 바뀌고, 브라우저 뒤로가기로도 이전 단계로 돌아간다.
- [ ] 지난주 요약에서 실패 분류를 확인한다 → "내부 N · 외부 N(관리됨 M)" 집계가 보인다.
- [ ] 마일스톤 안건에서 **유지 / 연기 / 주의로** 중 하나로 결정한다 → 그 자리에서 반영된다 (긴급 결정 카드와 같은 규칙).
- [ ] 결정 필요 섹션을 확인한다 → 미결 외부 변경·Action·결제·도움 요청이 모여 있다.
- [ ] 회의를 끝낸다 → 결정 내용이 **회의록 초안**으로 만들어진다 (저장은 관리자만).

## 2-6. 외부 변경 접수 (담당자·관리자)

- [ ] 데일리 오른쪽 위 **외부 변경 접수** 버튼을 누른다 → 접수 다이얼로그가 열린다 (담당 프로젝트가 없는 일반 팀원에게는 버튼이 보이지 않음).
- [ ] 프로젝트·제목을 적어 접수한다 → 데일리에 **진행 중 외부 변경 대응** 카드가 생기고, 5단계 진행 막대와 **영향 분석 마감까지 남은 시간**(24시간)이 표시된다.
- [ ] 단계를 순서대로 진행한다 (접수 → 영향 분석 → 재협의) → 순서 건너뛰기는 되지 않는다.
- [ ] **승인** 단계를 일반 담당자 계정으로 시도한다 → 진행되지 않는다 (승인은 관리자만).
- [ ] 관리자로 승인 후 **종결**한다 → 카드가 종결되고, 연결된 마일스톤에 "외부·관리됨" 회고가 자동으로 남는다.

## 2-7. 마일스톤 관리 (담당자·관리자)

- [ ] 반복·마일스톤 화면에서 마일스톤을 생성·수정한다 → 즉시 반영된다. 위험 표시(정상/주의/지연)를 바꾸면 색이 바뀐다.
- [ ] 마일스톤 칩을 눌러 **중요 마일스톤** 체크박스를 켠다 → 칩이 **옐로→오렌지 그라데이션**으로 바뀐다 (일반은 시안→옐로).
- [ ] 간트 뷰에서 내가 관리하는 마일스톤 칩을 **가로로 드래그**한다 → 기한이 이동되고, 토스트의 **[실행 취소]**를 누르면 원래 날짜로 복원된다.
- [ ] 기한을 넘겨 끝난 마일스톤의 **회고 남기기**를 누른다 → 내부/외부 원인·세부 유형·한 줄 메모를 저장할 수 있고, 회의 진행 모드의 지난주 집계에 반영된다.

## 2-8. 결제요청 (관리자 처리)

- [ ] 대기 건의 원형 체크 버튼을 누른다 → **완료**로 바뀌고 컨페티가 나온다 (입금 완료는 관리자만 가능).
- [ ] 취소된 건의 **되돌리기**를 누른다 → 대기로 돌아온다.
- [ ] **분류 관리**에서 분류를 추가·이름 변경·보관한다 → 등록 폼의 분류 목록에 반영된다.
- [ ] 관리자 계정에서는 모든 요청의 계좌·금액이 가려지지 않고 보이는지 확인한다 → 원래 값과 복사 버튼이 보인다.

## 2-9. 데일리 · OKR 관리 (관리자)

- [ ] AI 팀 요약의 **즉시 생성/다시 생성** 버튼을 누른다 → 요약이 (재)생성된다 (관리자만 가능, 수십 초 걸릴 수 있음).
- [ ] OKR 탭에서 분기 **목표(Objective)를 생성**한다 → 목록에 추가된다 (목표 생성은 관리자만).
- [ ] 목표 아래 **핵심결과(KR)를 추가**한다 → 측정 방식(수동/작업 자동/상태 체크)을 고를 수 있고, 상태 체크는 항목 1~7개와 항목별 '관리자 확인' 잠금을 걸 수 있다.
- [ ] 상태 체크 KR에서 **관리자 확인** 잠금이 걸린 항목을 일반 팀원 계정으로 본다 → 비활성(자물쇠)으로 표시되어 체크할 수 없다.

## 2-10. 프로젝트 — 담당자 권한 · AI 제안

- [ ] 내가 담당자인 프로젝트에서 **팀원의 작업** 상태를 바꾼다 → 반영된다 (프로젝트 담당자·관리자는 예외적으로 더 넓게 수정 가능 — 홀드·문제는 여전히 사유 필수).
- [ ] 간트 뷰(단일 프로젝트)에서 **AI 선행 연결 제안**을 실행한다 → 외부(Gemini) 전송 고지가 먼저 보이고, 제안은 건별 [연결] 버튼으로만 반영된다. 제안이 없으면 없다고 표시된다(날조 없음).

# 3부. 대표 추가

1부를 먼저 마친 뒤 진행하세요. 대표 계정이 관리자 권한을 겸하면 2부도 함께 진행합니다.

**준비물**: 대표 계정, 회사 Slack 로그인 상태
**예상 소요 시간**: 약 20분

- [ ] 로그인한다 → 홈이 열리고 제목이 **전사 현황**이다.
- [ ] 홈에서 **프로젝트 현황 · 업무 흐름 · 업무 부하 · 처리 대기** 칸을 확인한다 → 전사 기준 수치가 보이고, 업무 부하에는 전원이 포함된다.
- [ ] 홈에서 **클라이언트별 현황**과 **전 인원 히트맵**을 확인한다 → 대표 홈에만 표시된다.
- [ ] 성과 화면에서 **내 성과 · 팀 성과** 탭을 전환한다 → 팀 성과(기본)와 내 성과가 각각 계산되어 표시된다.
- [ ] 오전 9시 50분경 Slack **개인 DM**을 확인한다 → 오늘 할 작업·막힘·마감 임박·담당 마일스톤 위험 브리핑이 온다. 알릴 내용이 없는 날은 오지 않는 것이 정상이다. (개인 DM은 현재 대표만 수신하는 단계 롤아웃 상태)
- [ ] 다른 사람이 나에게 할 일을 배정하게 한다 (또는 관리자·회의록 확정으로 내게 작업 생성) → 생성 즉시 Slack 개인 DM으로 알림이 온다.
- [ ] 마일스톤이 위험 상태가 됐을 때 알림을 확인한다 → 담당자와 관리자(대표 포함)에게만 알림이 간다 (전원 아님 — 설계 의도, 설계 FAQ 참고).
- [ ] 팀 현황 리포트·결제요청 등 민감 수치가 잘 보이는지 훑는다 → 대표·관리자 권한 기준으로 계좌·금액·리포트가 가려짐 없이 표시된다.
