## 메뉴별 상세

### 1. 오늘 · `/today`
- **한 줄**: 내 하루를 시작하는 개인 화면 (팀원 기본 진입점)
- **주요 기능**
  - 요약 지표 5종 (진행중/문제/홀드/마감 임박/응답 대기 등)
  - 내 타임라인 — 내 작업 + 회사 일정 통합 (타인 비공개 일정은 `자리비움`으로 표시)
  - 작업 상태 변경 Sheet — 문제발생/홀드는 사유 폼 필수, 병합, 날짜·시간 변경
  - 자동 체크인 응답 패널 — 시작 시간 지난 작업에 자동 생성, 7지선다 응답
  - 주의 필요 패널 — 내 관련 문제/홀드, 나에게 온 도움 요청
  - 하루 마감 카드 — 오늘 완료 n / 미완료 목록 + [내일로] 이동
  - 빠른 등록(QuickAdd) — 자연어 입력 → 해석 → 확인 카드 → 작업 등록
  - 일정 충돌 변경 제안 — 시작 전 작업이 고정 일정과 겹치면 원클릭 이동 제안

### 2. Now · `/now`
- **한 줄**: 회의 Action과 캘린더 일정이 연결됐는지 확인하는 팀 운영표 (관리자·프로젝트 담당자 기본 진입점)
- **화면 제목**: "Now 운영표"
- **주요 기능**
  - 회의 Action + 캘린더 일정 통합 표
  - 필터 — 전체 / 내 항목 / 문제
  - 지표 5종, 담당자 미지정 항목 강조

### 3. 캘린더 · `/calendar`
- **한 줄**: 회사 일정, 작업, 마일스톤을 한 시간축에서 (단일 메뉴 — 별도 메뉴 금지, 뷰 전환은 화면 안에서)
- **주요 기능 (뷰 스위처)**
  - **기본형** — 시간축 격자(08–19시) + 마일스톤 밴드. 기간: 일간/주간/월간
  - **전체 멤버** — 사람 행 × 요일 열
  - **타임라인** — 방향(가로/세로) + 범위(시간별/1주간/2주간). 멤버·프로젝트 행, 막대/다이아몬드
  - 공통: 드래그로 작업·마일스톤 이동(회사 일정·자리비움은 읽기 전용), "수정됨" 배지, 마지막 뷰 기억·URL 반영

### 4. 팀 현황 · `/team`
- **한 줄**: 오늘 팀의 업무 흐름과 병목
- **주요 기능 (뷰 토글)**
  - **운영 보드** — 요약 5종, 사람별 오늘 시간표(충돌 배지), Attention Queue(문제/홀드/응답대기/도움요청), 일정 충돌 목록, 최근 변경
  - **스탠드업** — 멤버별 어제 완료/이월/오늘 예정/막힘 (아침 회의용)
  - **리포트** (관리자 전용) — 주간/월간 전체 현황·프로젝트별 완료·병목·부하 (점수화 없음)

### 5. 히트맵 · `/heatmap`
- **한 줄**: 멤버별 작업량 편차 — 평가가 아니라 업무 배분·병목 조정용
- **주요 기능**
  - 멤버 × 7일 격자, score(예상시간 + 문제/홀드/당일마감 가중), 강도 0–4 (색+수치 병기)
  - 셀 클릭 → 해당 날짜 전체 멤버 캘린더
  - 총 작업량 바 + 병목 수, 과부하/여유 요약(평균 대비)

### 6. 회의록 · `/meeting-notes`
- **한 줄**: Plaud 회의록을 업로드하고 Action 추출 대상으로 관리 (원문 항상 보존)
- **주요 기능**
  - MD 파일 업로드, 원문 미리보기, Action 추출 버튼
  - 공개 범위 — 팀 전체 / 관리자만 / 지정 인원만(restricted)
  - 지정 인원이 아닌 사람은 회의록·추출 Action 열람 불가

### 7. Action · `/action`
- **한 줄**: 회의록에서 추출된 Task 후보를 확정
- **주요 기능**
  - 후보 행에 담당자 지정 + 마감일 입력 → 둘 다 채우면 Task로 승격/생성
  - 보류/무시, 회의록 필터 칩
  - 담당자·마감 없는 Action은 Task로 자동 생성하지 않음(도메인 규칙)

### 8. 프로젝트 · `/projects`
- **한 줄**: 프로젝트 기준으로 마일스톤과 연결 작업을 추적
- **주요 기능**
  - 카드형 — 다음 마일스톤(D-day, at_risk 위험 배지), 진행률, 문제/홀드 목록, 담당자별 할당, 일정 변경 이력
  - 반복 업무 템플릿 등록/켜기·끄기 (주간/월간 정기 업무 자동 생성)

### 9. 결제 · `/payments`
- **한 줄**: 결제 요청을 등록하고 입금 처리 상태를 확인 (계좌번호·금액은 관리자·요청자에게만)
- **주요 기능**
  - 등록 폼, 마감 초과 대기 항목 상단 정렬(빨간 테두리)
  - 민감정보 마스킹 — 계좌/금액은 관리자·요청자 본인만 원본
  - 상태 버튼 권한 — 입금 완료=관리자만, 취소=관리자+요청자

## 🔄 메뉴 재편안 (2026-07-03 논의 중)

사용자 제안: 이름 변경 + 병합 + 신규 2개. 아래는 확정/미정 구분.

| **작업 목록** | 오늘 + Now | 🔀 병합 | ✅ **탭 전환(오늘/Now)** 방식 확정 |

### ✅ 확정된 결정
- 기존 "프로젝트"(마일스톤/반복템플릿) 화면은 **별도 메뉴로 유지**(삭제·통합 아님). 새 "프로젝트"는 캘린더를 가리키므로 이 화면의 새 이름 필요.
- "작업 목록"은 오늘 + Now를 **한 메뉴 안 두 탭(오늘 / Now)**으로 — 기존 두 화면 거의 그대로 유지하며 합침.

### ⏳ 착수 전 확정 필요 (3건)
1. **홈(전체현황) 내용** — 팀 전체 요약 대시보드(신규)? / 기존 관리자 리포트 승격? / 작업목록·일정 요약 조합?
2. **팀(팀원 카드) 범위** — 조회 전용 프로필 카드(이름·직급·이메일·전화·생년월일, que-user-info.md 필드)? / 카드+현재 업무 요약? / 카드+관리자 편집(→ users 스키마에 직급·전화·생년월일 등 컬럼 추가 필요)?
3. **기존 프로젝트 메뉴의 새 이름** — 예: "마일스톤", "프로젝트 상세" 등. (새 "프로젝트"=캘린더와 구분)

### 구현 메모 (확정 후)
- 이름 변경은 `apps/web/src/lib/menu.ts`의 `label` + 각 페이지 `PageHeader title` + `description`. **URL(`href`)은 유지 권장**(링크/쿠키/테스트 영향 최소화) — 즉 `/calendar`는 그대로 두고 라벨만 "프로젝트".
- 병합(작업목록=오늘+Now, 회의록+Action)은 신규 URL 하나에 탭 스위처(캘린더 뷰 스위처 패턴 재사용) + 기존 두 화면 컴포넌트 재배치.
- 신규(홈·팀)는 새 라우트 + 데이터 계층. 팀 편집까지 가면 users 스키마 확장(현재 email/password_hash만 추가돼 있음).
- **주의**: 이 재편은 `CLAUDE.md`의 "확정 메뉴 구조"를 바꾸는 것 — 착수 시 CLAUDE.md도 함께 갱신.

## 참고 (변경 시 유의)
- 메뉴 구조 자체(9개 개별 메뉴, 캘린더 단일화)는 `CLAUDE.md`에서 확정된 사항 — **이름만** 바꾸는 건 무방하나 구조 통폐합은 별도 논의.
- 메뉴명 변경 지점: `apps/web/src/lib/menu.ts`의 `label`. 화면 상단 제목(PageHeader `title`)과 사이드바 `description`도 함께 맞추면 일관됨.
- URL(`href`)까지 바꾸면 링크·쿠키·테스트 영향이 있으니, 표시명만 바꾸고 URL은 유지하는 걸 권장.
