# 캘린더 페이지 개발 프롬프트

```text
Que의 캘린더 페이지를 구현해줘.

참고:
- docs/que-product-plan.md의 캘린더 정의
- preview/calendar.html
- preview/app.js의 드래그 이동 동작

목적:
- 회사 일정, 개인 작업, 프로젝트 마일스톤을 주간 시간축에서 함께 확인하고 수정한다.

필수 UI:
- 기본형/가로형 옵션
- 주간/월간/일간 보기 토글
- 시간축 캘린더 그리드
- 회사 일정, Que 작업, 마일스톤 표시
- 작업 상세 패널 또는 Sheet
- 변경 로그
- 자연어 작업 추가 진입

드래그 정책:
- Que 작업과 프로젝트/캘린더 마일스톤은 드래그 가능
- 회사 캘린더 원본 일정과 비공개 자리비움은 드래그 불가
- 드롭 즉시 UI 반영
- 실패 시 원래 위치로 복구할 수 있는 구조
- 변경 로그에 누가, 무엇을, 어디로 이동했는지 기록

shadcn/ui:
- Tabs
- Button
- Badge
- Card
- Sheet
- Dialog
- ScrollArea
- Tooltip

반응형:
- FHD: 시간축 + 상세 패널을 함께 표시
- 태블릿 가로: 상세는 Sheet로 열거나 우측 패널 유지
- 태블릿 세로: 캘린더는 내부 가로 스크롤, 상세는 Sheet
- 시간 헤더와 요일 헤더는 sticky 처리

수용 조건:
- 회사 일정, 작업, 마일스톤이 시각적으로 구분된다.
- 드래그 가능한 항목과 불가능한 항목이 명확하다.
- 태블릿에서 드래그 또는 대체 이동 UI가 가능하다.
- lint/typecheck/build 통과.
```
