# 회의록 페이지 개발 프롬프트

```text
Que의 회의록 페이지를 구현해줘.

참고:
- docs/que-product-plan.md의 회의록 정의
- preview/meeting-notes.html

목적:
- Plaud Note에서 내보낸 Markdown 회의록을 업로드하고 Action 추출 대상으로 관리한다.
- 원문 보존과 출처 추적이 핵심이다.

필수 UI:
- MD 파일 업로드 영역
- 회의명 입력
- 프로젝트 선택
- 회의일 선택
- 참석자 입력/선택
- 공개 범위 선택
- 업로드된 회의록 목록
- 상태: 추출 대기, Task 생성, 확인 필요
- Markdown 원문 미리보기
- Action 화면 이동 버튼

파일 처리:
- MVP에서는 `.md` 파일만 받는다.
- 파일 선택 시 파일명, 크기, 업로드 가능 여부를 표시한다.
- Markdown body는 mock 저장 구조로 관리한다.
- 추후 실제 API 업로드로 바꾸기 쉽게 handler를 분리한다.

shadcn/ui:
- Card
- Button
- Input
- Textarea
- Select
- Badge
- ScrollArea
- Form
- Dialog
- Sonner

반응형:
- FHD: 업로드/입력 패널과 회의록 목록 2열, 아래 Markdown 미리보기
- 태블릿 가로: 2열 유지 가능
- 태블릿 세로: 업로드, 입력, 목록, 미리보기 순서로 스택
- 업로드 영역은 터치하기 쉽게 크게 유지

수용 조건:
- Plaud Note MD를 업로드하는 흐름이 명확하다.
- 업로드된 회의록에서 Action 추출 상태를 볼 수 있다.
- Markdown 원문이 손실되지 않고 미리보기된다.
- lint/typecheck/build 통과.
```
