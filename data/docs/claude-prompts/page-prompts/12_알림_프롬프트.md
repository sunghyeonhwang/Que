# 알림 페이지 개발 프롬프트

> **개발 후순위.** 알림 화면의 프리뷰는 추후 별도 제공 예정이다. 프리뷰를 받은 뒤 이 프롬프트를 프리뷰 기준으로 갱신하고 착수한다.

```text
Que의 알림 페이지를 구현해줘.

참고:
- docs/que-product-plan.md의 알림 정책

목적:
- 사용자가 처리해야 하는 알림을 모아보고, 불필요한 반복 알림을 줄인다.
- 알림은 읽는 것이 아니라 바로 처리 가능한 작업이어야 한다.

필수 UI:
- 알림 목록
- 필터: 전체, 체크인, 문제/홀드, 마감, 일정 충돌, 결제, Action
- 알림 상태: 새 알림, 읽음, 처리 완료
- 바로 처리 버튼
- 방해금지/나중에 답변 설정 진입

알림 유형:
- 작업 시작 체크인
- 문제발생/홀드 공유
- 도움 요청
- 마감 임박
- 일정 충돌
- 하루 요약
- 관련 작업의 시간/담당자/상태 변경
- 결제 요청 마감 임박/마감 초과
- 결제 요청 상태 변경
- 회의록 Action 후보 확인 필요

shadcn/ui:
- Card
- Badge
- Button
- Tabs
- Switch
- Select
- Sheet
- ScrollArea

반응형:
- FHD: 목록과 우측 상세/설정 패널
- 태블릿: 목록 우선, 상세는 Sheet
- 알림 row는 56px 이상으로 터치하기 쉽게 구성

수용 조건:
- 사용자가 어떤 알림을 처리해야 하는지 바로 안다.
- 처리 가능한 알림은 상세 페이지로 가지 않아도 주요 응답이 가능하다.
- 방해금지와 나중에 답변 흐름이 고려되어 있다.
- lint/typecheck/build 통과.
```
