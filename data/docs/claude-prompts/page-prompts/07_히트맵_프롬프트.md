# 히트맵 페이지 개발 프롬프트

```text
Que의 히트맵 페이지를 구현해줘.

참고:
- docs/que-product-plan.md의 히트맵 정의
- preview/heatmap.html

목적:
- 멤버별 작업량이 어느 날짜에 몰려 있는지 색상과 그래프로 확인한다.
- 업무 배분과 병목 조정을 위한 화면이며 개인 평가처럼 보이면 안 된다.

필수 UI:
- 상단 요약: 과부하 멤버, 여유 멤버, 총 예상 시간, 마감 임박, 문제/홀드
- 멤버별/요일별 작업량 히트맵
- 작업량 강도 범례
- 멤버별 총 작업량 막대 그래프
- 과부하/여유 멤버 요약
- 선택한 셀의 작업 목록 Sheet

계산 기준:
- taskCount
- estimatedHours
- issueCount
- holdCount
- dueSoonCount
- intensity

shadcn/ui:
- Card
- Badge
- Button
- ScrollArea
- Sheet
- Tooltip
- Progress

반응형:
- FHD: 히트맵과 우측 분석 패널 2열
- 태블릿 가로: 히트맵은 내부 스크롤, 분석 패널은 아래로 이동 가능
- 태블릿 세로: 히트맵 셀 최소 폭 유지, 사람 이름 sticky

수용 조건:
- 누가 바쁜지와 누가 여유 있는지 한눈에 보인다.
- 문제/홀드 가중치가 단순 작업 수와 구분된다.
- 특정 셀 클릭 시 해당 날짜/멤버 작업 목록으로 연결된다.
- lint/typecheck/build 통과.
```
