# 프로젝트 페이지 개발 프롬프트

```text
Que의 프로젝트 페이지를 구현해줘.

참고:
- docs/que-product-plan.md의 프로젝트 정의
- preview/project.html

목적:
- 프로젝트 기준으로 마일스톤, 연결 작업, 담당자, 문제발생 항목을 추적한다.

필수 UI:
- 프로젝트 개요
- 진행률
- 다음 마일스톤
- 캘린더 일정 그리드
- 연결된 작업 목록
- 문제/홀드 작업
- 담당자별 할당
- 일정 변경 이력
- 마일스톤/작업 드래그 이동

데이터:
- Project
- Milestone
- Task
- ChangeLog
- User

shadcn/ui:
- Card
- Badge
- Progress
- Tabs
- Button
- ScrollArea
- Sheet
- Tooltip

반응형:
- FHD: 프로젝트 개요와 마일스톤 요약을 상단 2열, 일정 그리드는 넓게
- 태블릿 가로: 일정 그리드 내부 가로 스크롤
- 태블릿 세로: 개요, 요약, 일정, 작업 목록 순서로 스택
- 드래그 대상은 최소 높이 44px 근처 유지

상호작용:
- 마일스톤 이동 시 dueAt 변경
- 연결 작업 이동 시 startAt/endAt 변경
- 변경 로그 즉시 추가
- 문제/홀드 클릭 시 상세 Sheet

수용 조건:
- 프로젝트 마감 위험과 다음 액션이 보인다.
- 마일스톤과 작업 이동이 캘린더 맥락과 연결된다.
- 태블릿에서 일정 그리드가 깨지지 않는다.
- lint/typecheck/build 통과.
```
