# 설정 페이지 개발 프롬프트

> **개발 후순위.** 설정 화면의 프리뷰는 추후 별도 제공 예정이다. 프리뷰를 받은 뒤 이 프롬프트를 프리뷰 기준으로 갱신하고 착수한다.

```text
Que의 설정 페이지를 구현해줘.

참고:
- docs/que-product-plan.md의 권한, 공개 범위, 알림 정책

목적:
- 개인 공개 범위, 알림, 캘린더 연동, 팀 멤버, 프로젝트 기본값을 관리한다.

필수 UI:
- 내 프로필
- 기본 공개 범위
- 개인 비공개 일정 표시 방식
- 알림 채널과 방해금지 시간
- 캘린더 연동 상태
- 팀 멤버 목록
- 프로젝트 기본 설정
- 결제 정보 마스킹 설정

권한:
- 일반 멤버는 본인 설정만 변경한다.
- 관리자는 팀 멤버, 프로젝트 기본값, 공개 범위 정책을 관리할 수 있다.

shadcn/ui:
- Form
- Input
- Select
- Switch
- Button
- Card
- Table
- Badge
- Tabs
- Separator

반응형:
- FHD: 설정 카테고리 목록 좌측, 상세 설정 우측
- 태블릿 가로: 2열 가능
- 태블릿 세로: 카테고리는 Tabs 또는 Select로 전환
- 폼 필드는 손가락 입력에 충분한 높이 유지

수용 조건:
- 공개 범위와 알림 설정이 명확하게 분리된다.
- 개인 비공개 일정의 보호 기본값이 안전하다.
- 관리자 전용 설정은 role에 따라 비활성/숨김 처리할 수 있는 구조다.
- lint/typecheck/build 통과.
```
