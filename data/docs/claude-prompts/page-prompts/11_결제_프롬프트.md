# 결제 페이지 개발 프롬프트

```text
Que의 결제 페이지를 구현해줘.

참고:
- docs/que-product-plan.md의 결제 정의
- preview/payment.html

목적:
- 멤버가 결제할 내용을 등록하고, 입금 후 완료/대기/취소 상태를 확인한다.

필수 UI:
- 결제 요청 등록 폼
- 입력 필드: 제목, 은행명, 계좌번호, 금액, 내용, 마감일, 분류
- 상태 요약: 전체 요청, 대기, 완료, 취소, 대기 금액
- 마감 임박/마감 초과 alert
- 결제 요청 목록
- 상태 버튼: 완료, 대기, 취소
- 상태 변경 로그 구조

데이터:
- PaymentRequest
- User
- ChangeLog

권한:
- 멤버는 본인이 필요한 결제 요청을 등록한다.
- 관리자 또는 결제 담당자는 입금 처리 여부를 확인하고 상태를 변경한다.
- 계좌번호와 금액은 추후 권한별 마스킹이 가능하게 구조화한다.

shadcn/ui:
- Form
- Input
- Textarea
- Select
- Button
- Badge
- Table 또는 Card list
- Alert
- Sonner

반응형:
- FHD: 등록 폼 좌측, 요청 목록 우측
- 태블릿 가로: 2열 유지 또는 목록 우선
- 태블릿 세로: alert, 등록 폼, 요청 목록 순서로 스택
- 상태 버튼은 3개가 한 줄에 안 들어가면 자연스럽게 줄바꿈

상호작용:
- 상태 버튼 클릭 시 Badge와 row 스타일 업데이트
- 완료는 green, 대기는 amber, 취소는 red
- 상태 변경 시 toast 또는 변경 로그에 기록

수용 조건:
- 결제 요청 입력 필드가 모두 있다.
- 완료/대기/취소 상태 변경이 명확하게 보인다.
- 마감 초과 대기 항목이 상단에 노출된다.
- lint/typecheck/build 통과.
```
