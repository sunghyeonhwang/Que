# Que 일괄 등록 CSV 샘플

작업(Task)·마일스톤(Milestone)을 CSV로 한 번에 등록하기 위한 템플릿.

- `tasks-sample.csv` — 작업(일정) 일괄 등록 (팀원 8명 전원 포함)
- `milestones-sample.csv` — 마일스톤 일괄 등록

> **id가 아니라 "이름"으로 채운다.** 담당자·클라이언트·프로젝트·도움 주는 사람 모두 **사람이 읽는 이름**으로 적고, 임포트 시 내부에서 실제 id로 변환한다. (예: `projectId=prj-summer`는 시스템 내부 id, `프로젝트명=여름 프로모션`은 표시 이름 — **둘은 다르다**. 샘플의 id가 슬러그라 비슷해 보였을 뿐.)
>
> ⚠️ 이름은 **Que에 이미 존재하는 값과 정확히 일치**해야 한다(앞뒤 공백은 무시). 프로젝트·클라이언트는 `/clients`, 담당자는 `/members`에서 확인. **없는 이름은 그 행이 실패**한다(임포트가 프로젝트·클라이언트·유저를 새로 만들지는 않음).

## 1. 작업 CSV (`tasks-sample.csv`)

| 컬럼 | 필수 | 값·형식 | 비고 |
|---|---|---|---|
| `title` | ✅ | 문자열(200자 이내) | 작업명 |
| `assigneeName` | ⬜ | 담당자 이름(아래 표) | **비우면 임포트 실행자 본인** |
| `clientName` | ⬜ | 클라이언트(거래처)명 | 동명 프로젝트 구분용. 프로젝트명이 유일하면 비워도 됨 |
| `projectName` | ⬜ | 프로젝트명 | 비우면 프로젝트 없음 |
| `startAt` | ⬜ | ISO8601 **오프셋 포함** | 예: `2026-07-14T10:00:00+09:00`(KST) |
| `endAt` | ⬜ | ISO8601 오프셋 포함 | `start ≤ end`. 한쪽만 있으면 같은 값으로 채움 |
| `priority` | ⬜ | `low` \| `normal` \| `high` | 비우면 `normal` |
| `estimatedHours` | ⬜ | 0보다 큰 숫자 | 예상 소요 시간(시간) |
| `description` | ⬜ | 문자열(2000자 이내) | 설명 |
| `status` | ⬜ | 아래 상태값 | 비우면 `scheduled`(예정) |
| `statusReason` | △ | 문자열(500자 이내) | **`on_hold`/`issue`면 필수**(사유). 그 외 상태는 비움 |
| `nextAction` | ⬜ | 문자열(500자 이내) | 다음 할 일(홀드/문제 시 권장) |
| **`helpUserNames`** | ⬜ | 담당자 이름, **여러 명은 `;`로**(최대 10) | **도움 주는(필요한) 사람**. 예: `이예진;황성진`. 홀드/문제 시 권장 |
| `recheckAt` | ⬜ | ISO8601 오프셋 포함 | 재확인 시간(홀드/문제 시 권장) |

**상태값(`status`)**: `scheduled`(예정) · `in_progress`(진행중) · `on_hold`(홀드) · `issue`(문제발생) · `needs_reschedule`(시간변경필요) · `done`(완료).

> **도움 주는 사람(`helpUserNames`)은 `on_hold`·`issue` 상태에서만 의미가 있다.** 도메인 규칙상 문제발생/홀드 전환은 **사유(`statusReason`) 필수** + 다음 액션·도움 필요한 사람·재확인 시간을 함께 받는다(그 외 상태에는 이 컬럼들을 비운다). 도움 대상은 `;`로 여러 명 지정한다(CSV 구분자 `,`와 충돌 방지).

**자동 지정(입력 불필요)**: `ownerId=`임포트 실행자 · `source=manual` · `visibility=team` · `id`. (임포트는 작업을 만든 뒤 `status`가 예정이 아니면 상태 변경까지 적용한다.)

## 2. 마일스톤 CSV (`milestones-sample.csv`)

| 컬럼 | 필수 | 값·형식 | 비고 |
|---|---|---|---|
| `clientName` | ⬜ | 클라이언트(거래처)명 | 동명 프로젝트 구분용 |
| `projectName` | ✅ | 프로젝트명 | Que에 실재해야 함(없으면 행 실패) |
| `title` | ✅ | 문자열 | 마일스톤명 |
| `dueAt` | ✅ | ISO8601 오프셋 포함 | 마감일시. 예: `2026-07-20T10:00:00+09:00` |
| `riskStatus` | ⬜ | `on_track`(정상) \| `at_risk`(주의) \| `late`(지연) | 비우면 `on_track`(정상) |

**권한**: 마일스톤 생성은 **관리자 또는 해당 프로젝트 담당자**만 가능(임포트 실행 계정이 그 권한이어야 함).

## 3. 담당자 이름 (assigneeName·helpUserNames용)

| 이름 | 역할 |
|---|---|
| 황성현 | 대표(admin) |
| 오승훈 | 관리(admin) |
| 황성진 | 관리(admin) |
| 송수용 | 관리(admin) |
| 박승환 | 팀원 |
| 이예진 | 팀원 |
| 김리원 | 팀원 |
| 이혜진 | 팀원 |

> 클라이언트·프로젝트명은 **본인 Que의 실제 값**을 쓴다(`/clients`에서 확인). 샘플의 멘딕스/에픽게임즈/그리프·여름 프로모션/결제 개선/CS 운영은 시드/데모 데이터라 실환경엔 다를 수 있다.

## 4. 임포트 방법 — 웹 화면 **설정 › 가져오기**

Que 웹의 **설정 › 가져오기** 탭에서 CSV를 올려 등록한다(구현 완료, 2026-07-10):

1. **설정 › 가져오기**에서 **작업** 또는 **마일스톤**을 선택하고, **샘플 CSV 받기**로 템플릿을 내려받아 채운다(이 폴더의 파일과 같은 형식).
2. **CSV 파일 선택** → **미리보기** — 각 행의 **이름을 실제 id로 해석**(담당자·도움주는사람=이름→유저, 프로젝트=클라이언트명+프로젝트명)하고 행별 **가능/오류 판정**을 보여준다. 모호(동명)·부재 행은 사유와 함께 오류.
3. **N건 등록** — 가능한 행만 core `createTask`/`createMilestone` 경유로 등록(도메인 규칙·권한·ChangeLog `via:"web"` 그대로 강제). `status`가 예정이 아니면 상태 변경까지 적용.
4. 알림: 담당자 **개인 DM은 발송하지 않는다**(대량 등록 방지). 단 **홀드/문제발생** 행은 팀 채널 병목 알림이 나간다.

> 📝 두 CSV의 **2행(`#`로 시작)은 각 컬럼 설명 가이드 행** — 가져오기 화면이 자동으로 무시하므로 지우지 않아도 된다. UTF-8로 저장(엑셀은 "CSV UTF-8"), 한 번에 최대 200행.
