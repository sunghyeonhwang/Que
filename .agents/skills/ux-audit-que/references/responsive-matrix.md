# 반응형·터치 매트릭스

Que 검수 해상도는 **태블릿 우선, FHD 보조**다. 아래 4개 해상도 각각에서 화면을 확인하고 매트릭스로 채점한다.

| 해상도 | 폭×높이 | 성격 |
| --- | --- | --- |
| FHD | 1920×1080 | 사무실 PC 기본 |
| 노트북 | 1366×768 | 세로 여유가 적음 — 잘림 취약 |
| 태블릿 가로 | 1024×768 | 사이드바 축소/collapsible 기준 |
| 태블릿 세로 | 768×1024 | 사이드바 → Sheet/Drawer/상단 전환 기준 |

## 각 해상도에서 점검하는 항목

- **A. 첫 화면 노출** — 핵심 정보(표/상태/오늘 할 일 등)가 스크롤 없이 첫 화면에 보이는가? FHD에서 특히.
- **B. 잘림** — 1366×768처럼 세로가 짧은 화면에서 메뉴·툴바·표 헤더가 잘리거나 겹치지 않는가?
- **C. 내부 스크롤 + sticky** — 표·캘린더·히트맵이 **페이지 전체를 밀지 않고** 자기 영역 안에서 스크롤되는가? header/label(사람 이름·날짜·시간)이 sticky로 고정되는가?
- **D. 사이드바 전환** — 태블릿 세로에서 사이드바가 본문을 밀어 레이아웃을 깨지 않고 Sheet/Drawer/상단·하단 내비게이션으로 전환되는가? 태블릿 가로에서 축소/collapsible 되는가?
- **E. 터치 대상** — 버튼·행·드래그·선택지가 최소 40px, 가능하면 44px 이상인가? 드래그가 불편한 태블릿을 위해 "날짜/시간 변경 Sheet" 같은 대체 경로가 있는가?

## 채점표 템플릿 (리포트에 그대로 붙인다)

```
| 항목 | FHD 1920 | 1366×768 | 태블릿 가로 1024 | 태블릿 세로 768 |
| --- | --- | --- | --- | --- |
| A 첫 화면 노출 | ✅/⚠️/❌ | … | … | … |
| B 잘림 없음 | | | | |
| C 내부 스크롤+sticky | | | | |
| D 사이드바 전환 | — | — | | |
| E 터치 40px+ | | | | |
```

- ✅ 통과 / ⚠️ 경미(개선 권장) / ❌ 실패(Blocker 후보) / — 해당 없음
- ❌가 하나라도 있으면 그 해상도는 "핵심 조작 불가" 여부를 반드시 서술한다. 특정 해상도에서 조작 불가는 Blocker다.

## 라이브 확인 팁

- 브라우저 자동화(Playwright MCP: `browser_resize`, Claude-in-Chrome: `resize_window`)로 창을 각 해상도로 맞추고 스크린샷을 남긴다.
- 표/캘린더는 실제로 스크롤해 보고, sticky가 유지되는지 스크롤 후 스크린샷으로 확인한다.
- 터치 대상 크기는 DevTools로 요소 높이를 재거나, 인접 클릭 대상 간격이 손가락으로 눌러 오탭할 여지가 있는지 본다.
